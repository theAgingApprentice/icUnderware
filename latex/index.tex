{\itshape An minimalistic \mbox{\hyperlink{class_logging}{Logging}} framework for Arduino-\/compatible embedded systems.}

Arduino\+Log is a minimalistic framework to help the programmer output log statements to an output of choice, fashioned after extensive logging libraries such as log4cpp ,log4j and log4net. In case of problems with an application, it is helpful to enable logging so that the problem can be located. Arduino\+Log is designed so that log statements can remain in the code with minimal performance cost. In order to facilitate this the loglevel can be adjusted, and (if your code is completely tested) all logging code can be compiled out.\hypertarget{index_autotoc_md19}{}\doxysubsection{Features}\label{index_autotoc_md19}

\begin{DoxyItemize}
\item Different log levels (Error, Info, Warn, Debug, Verbose )
\item Supports multiple variables
\item Supports formatted strings
\item Supports formatted strings from flash memory
\item Fixed memory allocation (zero malloc)
\item MIT License

The log library needs to be initialized with the log level of messages to show and the log output. The latter will often be the Serial interface. Optionally, you can indicate whether to show the log type (error, debug, etc) for each line.


\begin{DoxyCode}{0}

\DoxyCodeLine{begin(int level, Print* logOutput)}

The loglevels available are


\begin{DoxyCode}{0}
\DoxyCodeLine{* 0 -\/ LOG\_LEVEL\_SILENT     no output }
\DoxyCodeLine{* 1 -\/ LOG\_LEVEL\_FATAL      fatal errors }
\DoxyCodeLine{* 2 -\/ LOG\_LEVEL\_ERROR      all errors  }
\DoxyCodeLine{* 3 -\/ LOG\_LEVEL\_WARNING    errors, and warnings }

Newlines can be added using the {\ttfamily CR} keyword or by using the {\ttfamily ...ln} version of each of the log functions. The difference when using the {\ttfamily ...ln} is that the newline is placed after suffix, and only a single newline can be added.
